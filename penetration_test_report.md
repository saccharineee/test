## 1️⃣ 🔴 高危漏洞

### 1.1 会话固定（Session Fixation）

**漏洞描述**: 登录前后 session cookie **完全相同**。攻击者可先获取一个有效 session，诱骗受害者使用该 session 登录，登录后 session 不变，攻击者可共享受害者 session。

**复现**:
```python
# 攻击者获取session
s = requests.Session()
s.get('http://target/login')
malicious_session = s.cookies['session']

# 攻击者将malicious_session发给受害者
# 受害者用该session登录
# 攻击者可以用自己的session访问受害者的账户
```

**影响**: 可能导致账号被劫持
**修复建议**: `session.regenerate()` 或 `del session['username'] + session.clear()` 后重新生成 session

## 2️⃣ 🟠 中危漏洞

### 2.1 用户枚举（User Enumeration）

**漏洞描述**: 已存在用户和不存在用户的登录失败响应 **长度不同**（响应渲染时 hashlib 计算时间差异导致微小差异），可通过响应内容差异枚举有效用户名。

**现状**: 当前版本错误信息相同，但速率限制被绕过时仍可区分

**影响**: 辅助暴力破解
**修复建议**: 确保两种情况的响应完全一致（包括响应时间）

### 2.2 无速率限制绕过

**漏洞描述**: 速率限制基于进程内字典，**重启即重置**。攻击者可在每次限速后等待15分钟继续爆破。

**影响**: 暴力破解时间成本降低
**修复建议**: 使用 Redis 或持久化存储速率计数

### 2.3 Server 信息泄露

**漏洞描述**: HTTP响应头泄露：
```
Server: Werkzeug/3.1.8 Python/3.13.12
```
暴露了 Werkzeug 和 Python 的具体版本号。

**影响**: 辅助漏洞利用（已知版本CVE搜索）
**修复建议**: 使用反向代理隐藏版本信息，或自定义响应头

## 3️⃣ 🟡 低危漏洞

### 3.1 无HTTPS

**漏洞描述**: 仅HTTP通信，密码/会话明文传输。

**影响**: 中间人攻击可窃取会话和密码
**修复建议**: 生产环境必须启用HTTPS

### 3.2 无账户锁定

**漏洞描述**: 仅限速不锁定，攻击者可长期低速持续爆破。

**影响**: 长期持续攻击
**修复建议**: 多次失败后临时/永久锁定账户

### 3.3 CSRF Token 重放保护不足

**漏洞描述**: 登录后 CSRF token 更新，但未验证 token 与 session 的绑定关系。

**影响**: 复杂度低
**修复建议**: 绑定 token 到用户 IP 或 user-agent
